Update all imports in `src/main.py` and `src/fs_ui/orchestrator.py` to point to the new runner file.

- Move `run_cleaning_pass`, `run_grouping_pass`, `run_routing_pass`, and `run_generation_pass` from `src/main.py` to the new file.
- Extract any imports needed by these functions (like `json`, `fitz`, `Pipeline`, `DocumentGroup`, `setup_logging`, `set_verbosity`, etc.) and ensure they are imported at the top of `src/pipeline/runner.py`.

### Task 2: Update imports and CLI wiring in `src/main.py`

- Import them from `src.pipeline.runner`.
- Verify argument references and imports remain valid.

### Task 3: Update imports in `src/fs_ui/orchestrator.py`
- Change imports of these runner functions from `src.main` to `src.pipeline.runner`.

### Task 4: Verify test suite execution
- Run the full pytest suite to verify everything still compiles, imports cleanly, and passes successfully.
