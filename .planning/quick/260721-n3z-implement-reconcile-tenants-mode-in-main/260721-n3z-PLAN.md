## Description
Implement a new CLI command `reconcile --tenants` in `main.py` that allows retroactively updating tenant routing based on an updated `_tenants.yaml` configuration. This will adjust the intermediate state JSONs and move previously generated PDFs to match the new tenant assignments.

## Tasks

### Task 1: Add `reconcile` Subparser to CLI
- Update `src/main.py` -> `get_parser()` to register a new `reconcile` subcommand.
- Add `--tenants` flag, `--dry-run` flag, and `target_dir` positional argument to the `reconcile` subparser.
- Wire the command in `main()` to invoke a newly defined `run_reconcile_mode(args)` function.

### Task 2: Implement State Updates (`_1`, `_2`, `_3` JSONs)
- Create the `run_reconcile_mode` function. 
- Read `.source_files/[house_id]_tenants.yaml` to obtain the latest tenant names and their `start_date`/`end_date` timeline.
- Read `.source_files/[house_id]_1_cleaned.json`, `_2_grouped.json`, and `_3_routed_and_finalized.json`.
- Parse dates to determine the correct tenant for each document.
- Iterate through documents in `_1_cleaned.json` and `_2_grouped.json` and update `canonical_tenant`.
- Update `target_folder` and `file_name` paths in `_3_routed_and_finalized.json` to reflect the updated tenant assignment.
- Save the updated JSONs back to the disk.

### Task 3: Move PDFs and Clean Up Empty Directories
- For each document modified in Task 2, determine its old file path (from existing `_3_routed...` state) and new file path.
- Move the physical PDFs on disk from their old locations to the new locations (handling directory creation if the new tenant folder doesn't exist yet).
- Iterate through the `target_dir` subdirectories and delete any old tenant folders that are now empty as a result of the moves.
- Output a summary of moved files and deleted directories.
