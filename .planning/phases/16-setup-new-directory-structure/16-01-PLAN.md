# Phase 16: Setup New Directory Structure

## Overview
This plan focuses on reorganizing the `src/` directory into a logical, modular structure: `core`, `utils`, `tenant_config`, `grouping`, `timeline`, and `routing`. The goal is to preserve all existing functionality while making the directory structure flatter and more modular.

## Task 1: Create New Directories
- Create the following directories inside `src/` if they don't already exist:
  - `src/utils`
  - `src/tenant_config`
  - `src/timeline`
  - `src/grouping` (extract from `src/processing/grouping`)
  - `src/routing` (extract from `src/processing/routing`)

## Task 2: Move and Reorganize Files
1. **Core (`src/core`)**:
   - Keep existing `src/core/` files (`config.py`, `exceptions.py`, `indexing.py`, `schemas.py`, `ui.py`, `utils.py`).
2. **Utils (`src/utils`)**:
   - Move `src/fs_utils.py` to `src/utils/fs.py` (or keep as `fs_utils.py`).
   - Move `src/logger.py` to `src/utils/logger.py`.
3. **Tenant Config (`src/tenant_config`)**:
   - Move `src/cleaning/tenants.py` to `src/tenant_config/tenants.py`.
4. **Grouping (`src/grouping`)**:
   - Move all contents of `src/processing/grouping/` to `src/grouping/`.
5. **Timeline (`src/timeline`)**:
   - Move `src/cleaning/dates.py` to `src/timeline/dates.py`.
   - Move `src/cleaning/phase.py` to `src/timeline/phase.py`.
   - Move `src/processing/organizer/` contents to `src/timeline/`.
6. **Routing (`src/routing`)**:
   - Move all contents of `src/processing/routing/` to `src/routing/`.
7. **Processing / Pipeline**:
   - Move `src/processing/pipeline.py` and `src/processing/visualizer.py` to appropriate locations (e.g., `src/core/` or keep in a flattened `src/pipeline/`).

## Task 3: Update Imports
- Perform a project-wide search and replace to update all import paths across the codebase.
- Ensure that `src/main.py` imports correctly from the new structure.
- Run `pytest` and verify no `ImportError` or `ModuleNotFoundError` is raised.

## Verification
- Run the full test suite to ensure functionality remains identical.
- Check that the application starts up and processes sample data without errors.
