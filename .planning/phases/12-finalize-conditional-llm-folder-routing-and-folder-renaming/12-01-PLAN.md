# Phase 12 Plan 01: Unify Routing Configuration and Final Validation

**One-liner:** Centralize direct routing configuration and perform a final system-wide audit of Arabic folder naming to close out the routing feature set.

## Context
Phase 11 implemented the core conditional routing logic. However, a redundancy exists between `src/processing/routing/config.py` (which defines `DIRECT_ROUTED_CATEGORIES`) and `src/processing/routing/router.py` (which uses a local `DIRECT_ROUTING_MAP`). This inconsistency increases the risk of routing errors if categories are updated in one place but not the other.

## Objectives
- Eliminate configuration redundancy by centralizing all direct routing mappings in `config.py`.
- Ensure strict consistency between classification categories and routing keys.
- Perform a final "zero-English" audit for folder names in the source code.
- Verify that the unified configuration preserves all existing routing behavior.

## Tasks

### 1. Centralize Direct Routing Configuration
- [x] **Move Mapping to Config:** Transfer the `DIRECT_ROUTING_MAP` from `src/processing/routing/router.py` to `src/processing/routing/config.py`.
- [x] **Standardize Keys:** Ensure the keys in the map match the normalized (lowercase) category strings used in `route_document`.
- [x] **Remove Redundancy:** Remove the now-redundant `DIRECT_ROUTED_CATEGORIES` set from `config.py`, as the map keys themselves define the direct-route set.
- [x] **Update Router:** Modify `src/processing/routing/router.py` to import and use the centralized map.

### 2. Final System-Wide Audit
- [x] **English-to-Arabic Scan:** Run a `grep` search across `src/` to identify any remaining hardcoded English folder names (e.g., "Contracts", "ID Cards") that should have been migrated to Arabic.
- [x] **Naming Consistency Check:** Verify that all folder references in `src/processing/organizer/core.py` and other integration points use the mapping from `config.py`.

### 3. Validation & Verification
- [x] **Unit Test Verification:** Run `pytest tests/test_routing.py` and `tests/test_routing_logic.py` to ensure direct routing still functions correctly.
- [x] **Regression Test:** Verify that the "Others" double-check flow and constrained routing for Forms/Letters remain intact.
- [x] **E2E Validation:** Run `tests/test_e2e.py` to confirm full pipeline integrity.

## Definition of Done
- [x] `DIRECT_ROUTING_MAP` exists only in `config.py`.
- [x] `router.py` contains no local routing maps.
- [x] No hardcoded English folder names remain in `src/`.
- [x] All routing and E2E tests pass.
