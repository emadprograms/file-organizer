# Plan 11-02: Constrained LLM Routing and "Others" Double-Check

## Objective
Reduce LLM hallucinations by constraining the available folder choices for specific categories and implementing a rigorous double-check mechanism for the "Others" category.

## Research & Analysis
- **Constrained Categories:** `Forms` and `Letters`.
- **Others Category:** `OTHER_LETTERS`.
- **Current Prompt:** Uses a static list of all folders and a generic "Allowed Folders" section.
- **Review Feedback:** Need strict type-level enforcement of constrained lists, an escape hatch for misclassifications, and robust hallucination handling during retries.

## Tasks

### 1. Implement Constrained Prompting
- Modify `route_document` in `src/processing/routing/router.py` to filter `allowed_folders` based on the document category:
    - For `Forms` (e.g., `BASIC_DETAILS`, `KEY_HANDOVER`, `MAINTENANCE`): Only present folders associated with forms.
    - For `Letters` (e.g., `AMAR_TAKHSEES`, `RENT_DEDUCTION`, `NOTIFICATIONS`): Only present folders associated with letters.
- **Escape Hatch:** Add a `"None of the above"` option to the constrained lists for both Forms and Letters. If selected, the document should be routed to the `Others` flow (`OTHER_LETTERS`).
- **Strict Enforcement:** Use a dynamic Pydantic `Enum` or inject the current `allowed_folders` into the `RoutingResponse` schema description to ensure the LLM output is validated against the constrained list at the type level.
- **Retry Trigger:** Ensure that any validation failure (e.g., LLM returns a folder not in the constrained list) explicitly triggers the standard retry mechanism.

### 2. Implement "Others" Double-Check Logic
- For documents categorized as `OTHER_LETTERS` (or those that hit the "None of the above" escape hatch):
    - **Step 1:** Prompt the LLM with all 13 Arabic folders.
    - **Step 2:** If the result is `رسائل متنوعة` (Miscellaneous Letters), accept immediately.
    - **Step 3:** If the result is any other folder, perform a second LLM call to confirm: "You previously selected [Folder]. Are you sure this is the best fit, or should it be 'رسائل متنوعة'?"
    - **Step 4 (Hallucination Handling):** If the second call returns a folder name that is NOT in the definitive Arabic mapping (hallucination), automatically fallback to `رسائل متنوعة`.
    - **Step 5:** If the second call confirms the same folder, accept it. Otherwise, default to `رسائل متنوعة`.
    - **Limit:** Implement a maximum of 2 retry iterations before defaulting to `رسائل متنوعة`.

### 3. Update Validation Logic
- Ensure the Pydantic `RoutingResponse` validation context is updated to use the constrained `allowed_folders` list for each request.

## Verification Plan
- **Unit Tests:** 
    - Create test cases in `tests/test_routing.py` for `Forms` and `Letters` categories:
        - Mock LLM returning a folder OUTSIDE the constrained list $\rightarrow$ verify it is rejected and retried.
        - Mock LLM returning `"None of the above"` $\rightarrow$ verify it routes to the `Others` flow.
    - Create test cases for the `Others` flow:
        - One where the LLM picks `رسائل متنوعة` initially.
        - One where the LLM picks another folder and confirms it in the second call.
        - One where the LLM picks another folder and changes its mind in the second call $\rightarrow$ default to `رسائل متنوعة`.
        - **NEW:** Mock LLM returning a hallucinated/invalid folder name during the second call $\rightarrow$ verify it fallbacks to `رسائل متنوعة`.
        - **NEW:** Mock LLM failing to converge after 2 retries $\rightarrow$ verify default to `رسائل متنوعة`.
- **Log Check:** Verify that "Double-check" steps, "None of the above" triggers, and hallucination fallbacks are clearly logged in the decision trace.
