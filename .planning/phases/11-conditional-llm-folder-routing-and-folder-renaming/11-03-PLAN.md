# Plan 11-03: System-Wide Integration and Validation

## Objective
Ensure that the new Arabic folder mapping is applied consistently across the entire application and that the routing system is robustly verified.

## Research & Analysis
- **Folder References:** Folder names may be used in `src/processing/organizer/core.py`, `src/processing/routing/config.py`, and various tests.
- **Existing Tests:** `tests/test_routing.py`, `tests/test_routing_logic.py`, and `tests/test_e2e.py` rely on folder name assertions.
- **Review Feedback:** Need to verify if any external API contracts, frontend expectations, or CI/CD scripts rely on the old folder names before implementing system-wide renaming.

## Tasks

### 1. Analyze Downstream Dependencies
- Search the codebase and any configuration files for usages of folder names that might be expected by external systems or scripts.
- Verify if there are any hardcoded paths in CI/CD pipelines or deployment scripts that would be broken by switching to Arabic folder names.
- Document any dependencies that require special handling or exceptions.

### 2. System-Wide Folder Name Update
- Search the entire `src/` directory for any hardcoded English folder names or IDs from the old routing system (e.g., `1_requests_and_applications`).
- Replace them with references to the new Arabic mapping defined in `src/processing/routing/config.py`.
- Ensure that `src/processing/organizer/core.py` correctly uses the Arabic names when creating target directories.

### 3. Comprehensive Routing Test Suite Update
- Update all assertions in `tests/test_routing.py` and `tests/test_routing_logic.py` to expect Arabic folder names.
- Update `tests/test_core.py` and any other tests that validate final destination file paths to reflect the new Arabic folder structure.
- Add edge-case tests for the constrained routing:
    - Documents that barely fit into a constrained category.
    - Documents that the LLM tries to push into a forbidden folder.

### 4. E2E Validation
- Run `tests/test_e2e.py` to ensure that a full document processing run results in files being moved to the correct Arabic folders.
- Verify that the final directory structure on disk matches the intended Arabic mapping.

## Verification Plan
- **Dependency Audit:** Complete the analysis in Task 1 and confirm no critical breaking changes are introduced to external contracts.
- **Static Analysis:** Run `grep` (or equivalent) to ensure no old English folder IDs remain in the `src/` directory.
- **Test Execution:** Run `pytest tests/test_routing.py tests/test_routing_logic.py tests/test_e2e.py tests/test_core.py`.
- **Manual Audit:** Inspect the output folder structure of a sample run to confirm Arabic names are used and correctly assigned.
