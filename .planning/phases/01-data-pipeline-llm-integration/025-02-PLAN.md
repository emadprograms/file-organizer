# Plan 025-02: Entity Resolution LLM Pass (Pass 1.5)

## Objective
The current Python deterministic grouping fails to map English names to Arabic equivalents, and fails to realize that family members (like wives) who appear on random letters (like EWA) belong to the primary tenant's timeline. We will solve this by implementing Pass 1.5: a single, fast LLM call that receives the entire chronological log of extracted names and outputs a JSON mapping of all raw names to a Canonical Primary Tenant.

## Tasks

### 1. Update Schemas for Pass 1.5
- In `src/schemas.py`, add a new Pydantic schema `class EntityResolutionMapping(BaseModel)` that expects `mapping: dict[str, str]` representing a map of `{ "Raw Name": "Canonical Arabic Primary Tenant Name" }`.

### 2. Implement LLM Resolution Call
- In `src/llm.py`, add a new method `resolve_entities(self, raw_pages_log: str) -> dict[str, str]`.
- Build a system prompt that explains:
  - You are receiving a chronological log of documents `[Category, Name, Date]` for a single house.
  - 1. Identify the Primary Tenants (Head of Household).
  - 2. Map all English/Arabic/abbreviated variations into one Canonical Name.
  - 3. Identify family members (wives, children) who appear in the log, and map them to their respective Primary Tenant.
  - Return the JSON mapping using the `EntityResolutionMapping` schema.

### 3. Update Pipeline Architecture (Pass 1.5)
- In `src/pipeline.py`, inside `process_pdf`, between Pass 1 (vision extraction) and Pass 2 (tenant grouping):
  - Build a string log of all extracted pages: e.g., `Page 12: ewa_related_letters | امنه_الله_بخش_هاشم_رحيم | 26/07/2017`.
  - Pass this log to `self.client.resolve_entities()`.
  - Receive the `canonical_mapping` dictionary.

### 4. Update the Grouping Loop (Pass 2)
- In `src/pipeline.py`, inside the `raw_pages` loop:
  - Remove the brittle `_is_same_person` fuzzy matching.
  - Whenever you need to identify the tenant of a document, look up `page.resident` in the `canonical_mapping` dictionary.
  - Use the mapped canonical name as the `page_primary_tenant`.
  - Maintain the existing category rules (`amar_takhsees` independent, `personal_details` inherits the active canonical timeline).

## Context Notes
- This approach uses LLM context exactly where it excels (Entity Resolution across a timeline) and relies on Python code where it excels (Deterministic State Machine Document Grouping).
