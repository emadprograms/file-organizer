# Plan 03 â€” Sequential Pipeline & Sliding Window Accumulator

## Goal

Replace the parallel `ThreadPoolExecutor` pipeline with a sequential page processor that implements the continuation-based grouping described in D-03. The pipeline feeds page images (not text) to `GemmaClient.classify_page()` via multimodal vision, groups consecutive continuation pages into document segments, and emits completed segments with their metadata.

## must_haves

- [ ] `Pipeline.process_pdf()` processes pages sequentially (no ThreadPoolExecutor)
- [ ] Pipeline uses `extract_pages_as_images()` to get page images for Gemma's multimodal vision
- [ ] `previous_summary` context is passed between groups so Gemma can detect continuations
- [ ] Pages accumulate into a group until `is_continuation=False` starts a new group
- [ ] Each emitted document contains `start_page`, `end_page`, `house_number`, `resident`, `category`
- [ ] Pipeline returns `list[DocumentGroup]` dataclass, not raw dicts
- [ ] Test stubs `test_continuation_detection` and `test_sliding_window` are unskipped and implemented

## Artifacts this phase produces

### New dataclass in `src/schemas.py`
- `class DocumentGroup` â€” fields: `start_page: int`, `end_page: int`, `house_number: str`, `resident: str`, `category: Category`

### Refactored file: `src/pipeline.py`
- Removes `import concurrent.futures`
- `Pipeline.__init__(self, api_keys, delay_between_pages)` â€” stores `delay_between_pages`, no `max_workers`
- `Pipeline.process_pdf(self, pdf_path: str) -> list[DocumentGroup]` â€” sequential loop:
  1. Iterates `ingestor.extract_pages_as_images(pdf_path)`
  2. Maintains `current_group_start: int`, `current_classification: PageClassification`, `previous_summary: str`
  3. Calls `client.classify_page(image_bytes, previous_summary)`
  4. If `is_continuation=True`: extends current group's `end_page`
  5. If `is_continuation=False`: emits `DocumentGroup(start_page, end_page, house_number, resident, category)`, resets accumulator, sets `previous_summary` to a brief string describing the completed group
  6. After loop ends, emits final group
  7. Returns `list[DocumentGroup]`

### Updated tests in `tests/test_llm.py`
- `test_continuation_detection()` â€” unskipped, tests that pipeline correctly groups continuation pages
- `test_sliding_window()` â€” unskipped, renamed to `test_previous_summary_context()`, tests that previous_summary context passes between groups
