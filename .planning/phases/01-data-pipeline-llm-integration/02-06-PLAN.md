# Plan 02-06: Two-Pass Global Grouping Architecture

## Objective
Refactor the pipeline to solve the "family fragmentation" issue. The system will shift from a 1-pass sequential sliding window to a 2-pass global architecture. Pass 1 extracts raw facts from images. Pass 2 feeds the global list of facts to a text-only LLM prompt to identify the core family and group related pages perfectly.

## Steps

### Step 1: Update Schemas
- **File:** `src/schemas.py`
- **Action:** 
  - Remove `is_continuation` from `PageClassification`.
  - Ensure `DocumentGroup` accurately reflects the desired output of the second pass (start_page, end_page, house_number, primary_tenant_name, category).

### Step 2: Implement Pass 2 Global Grouping Logic in LLM Client
- **File:** `src/llm.py`
- **Action:**
  - Remove `active_summary` and `previous_summary` from `classify_page` and its prompts.
  - Create a new method `group_document_pages(self, pages: list[PageClassification]) -> list[DocumentGroup]`.
  - In `group_document_pages`, build a text-only prompt that includes the JSON serialization of all extracted pages.
  - Instruct the LLM to identify the main family tenants and merge consecutive pages (especially `personal_details` and `basic_details`) belonging to the same family unit into single `DocumentGroup` clusters. Note the `amar_takhsees` exception.
  - Use `gemma-4-31b-it` (text only) and return a list of `DocumentGroup` objects via `response_schema`.

### Step 3: Refactor Pipeline to Use Two Passes
- **File:** `src/pipeline.py`
- **Action:**
  - In `process_pdf`, remove the sliding window variables (`active_summary`, `previous_summary`, `current_classification`).
  - **Pass 1:** Iterate through all images, calling `classify_page` for each, and append the results to a `raw_pages` list.
  - **Pass 2:** Pass the complete `raw_pages` list to `self.client.group_document_pages()`.
  - Return the grouped documents.
