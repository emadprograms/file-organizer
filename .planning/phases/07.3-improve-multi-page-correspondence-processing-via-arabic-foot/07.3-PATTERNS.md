# Phase 07.3: Improve multi-page correspondence processing via Arabic footer - Patterns

## Files to be Modified

### `src/pipeline.py`

#### Modification 1: Pass 1 Pipeline - LLM Skip Mechanism via Vision OCR
- **Role:** Quickly detect Arabic continuation footers (e.g., "الصفحة 2", "1 من 10 الصفحة") using Mac's native Vision framework on the bottom 10% of the page to bypass expensive LLM calls.
- **Data Flow:** `process_single_page(page_info)` receives `image_bytes` -> Convert `image_bytes` for Vision framework -> Execute `VNRecognizeTextRequest` on bottom 10% -> Regex/String match for Arabic footer -> If match is found, cache and return a dummy `PageClassification` with `is_continuation=True` -> Else, fall back to the standard `client.classify_page(image_bytes)` call.
- **Closest Analog:** The existing pre-check for completely blank pages that skips the LLM call based on byte size threshold.
- **Concrete Code Excerpt (Analog):**
  ```python
  # Pre-check for completely blank pages using file size threshold (15KB for 150 DPI)
  if len(i_bytes) < 15000:
      print(f" Page {p_idx} is blank (size {len(i_bytes)} bytes). Skipping LLM.")
      res = PageClassification(
          category=Category.OTHER_LETTERS,
          residents=["NONE"],
          date="NONE",
          house_number="UNKNOWN"
      )
      with cache_lock:
          cached_pages[str(p_idx)] = res.model_dump()
          temp_cache_file = f"{cache_file}.tmp"
          with open(temp_cache_file, "w", encoding="utf-8") as f:
              json.dump(cached_pages, f, ensure_ascii=False, indent=2)
          os.replace(temp_cache_file, cache_file)
      return (p_idx, res)
  ```

#### Modification 2: Pass 2 Grouping Logic - Lookahead Window
- **Role:** Group consecutive pages into documents robustly by using a lookahead window, tolerating missing or misprinted footers on intermediate pages as long as the sequence continues. It also needs to bypass strict categorization checks for dummy categories generated by the Pass 1 skip logic.
- **Data Flow:** `_group_pages_into_documents(raw_pages, canonical_mapping_clean)` iterates over pages -> Maintain sequence state -> Look ahead to page `i+1` when evaluating page `i` -> If current page lacks `is_continuation` but next page has it, or if it is part of an ongoing sequence, merge into the current document group.
- **Closest Analog:** The existing `merge_condition` in `_group_pages_into_documents` which strictly groups based on identical categories and primary tenants, and handles the existing `is_continuation` property.
- **Concrete Code Excerpt (Analog):**
  ```python
  merge_condition = False
  if group_list and group_list[-1].category == page.category and group_list[-1].primary_tenant == page_primary_tenant:
      if getattr(page, 'is_continuation', False):
          merge_condition = True
      elif page.date != "NONE" and group_list[-1].dates and group_list[-1].dates[-1] == page.date:
          merge_condition = True

  if merge_condition:
      group_list[-1].end_page = page_index
      if page.date != "NONE":
          group_list[-1].dates.append(page.date)
  else:
      group_list.append(DocumentGroup(
          start_page=page_index,
          end_page=page_index,
          house_number=page.house_number,
          primary_tenant=page_primary_tenant,
          category=page.category,
          dates=[page.date] if page.date != "NONE" else []
      ))
  ```
