# Phase 07.3 Research: Improve multi-page correspondence processing via Arabic footer

## Question
**What do I need to know to PLAN this phase well?**

To plan this phase effectively, you must understand how to integrate Mac's native Vision framework directly into the Pass 1 concurrent pipeline to detect Arabic footers, bypassing expensive LLM calls, and how to update the Pass 2 sequential logic to group these documents using a lookahead window.

## Core Objectives & Requirements
- **LLM Skip Mechanism (D-03)**: Utilize Mac's native Vision framework (`pyobjc-framework-Vision`) on the bottom 10% of the page to detect footers, completely skipping the heavy LLM call for continuation pages.
- **Pattern Matching Strictness (D-01)**: Use flexible string matching or regex to tolerate OCR imperfections when looking for Arabic footers (e.g., "1 من 10 الصفحة" or "الصفحة 2").
- **Grouping Assurance (D-02)**: If page N is missing a footer but N+1 has it, tolerate the miss and group N anyway (lookahead window). A complete absence breaks the sequence.

## Technical Context & Implementation Details

### 1. Mac Vision Framework Integration (`src/pipeline.py`)
- **Framework Setup**: The logic requires `pyobjc-framework-Vision` and `pyobjc-framework-Quartz` (or `CoreMedia`) to process the raw image bytes generated by `PdfIngestor`.
- **Extraction Logic**: Inject a fast OCR check into `process_single_page` *before* the `client.classify_page(image_bytes)` call.
- **Region of Interest**: Utilize `VNRecognizeTextRequest`. Crucially, set its `regionOfInterest` to the bottom 10% (e.g., `CGRectMake(0.0, 0.0, 1.0, 0.1)` in normalized coordinates where origin is bottom-left) to minimize OCR time and avoid false positives from the body text.
- **Arabic Support**: Ensure `request.setRecognitionLanguages_(["ar"])` is called to accurately read the Arabic numerals and text.

### 2. Pass 1 Pipeline Modifications
- **Detecting Continuations**: Analyze the extracted footer text for patterns indicating page > 1 (e.g., "2 من", "الصفحة 2", "3 من 4").
- **Skipping LLM**: If a continuation page is detected, do **not** call `client.classify_page`. Instead, immediately cache and return a dummy `PageClassification` with `is_continuation=True`, `category=Category.OTHER_LETTERS`, and `residents=["UNKNOWN"]`.
- **Thread Safety**: Pass 1 processes pages concurrently using a `ThreadPoolExecutor`. The Vision OCR instantiation must be stateless and thread-safe. Pages will be processed out of order, so you cannot rely on inheriting values from the "previous" page during Pass 1.

### 3. Pass 2 Grouping Logic Modifications (`src/pipeline.py`)
- **Bypassing Strict Categorization**: The `_group_pages_into_documents` function must be updated. Currently, it strictly enforces `group_list[-1].category == page.category` before allowing `is_continuation` to merge. Since Pass 1 now returns dummy categories for skipped pages, this strict check must be bypassed if a continuation sequence is active.
- **Lookahead Window (D-02)**: Modify the iteration logic (e.g., using `enumerate` on `raw_pages`) to allow looking ahead. If `page[i]` lacks the `is_continuation` flag, but `page[i+1]` has it, safely merge `page[i]` into the current document group to handle missing or misprinted footers on a single page.
- **Sequence Breaking**: A sequence is broken (and a new document group initialized) only when multiple consecutive pages fail the lookahead check, indicating a genuine end to the correspondence.

## Validation Architecture

1. **Vision OCR Component Testing**: Ensure the Mac Vision framework successfully targets the bottom 10% and extracts Arabic footer text from sample PNG bytes.
2. **Regex Pattern Tolerance**: Validate that the pattern matching correctly identifies variations and handles missing spaces due to typical OCR flaws.
3. **Pass 1 LLM Skip Verification**: Assert that when a footer indicates page > 1, `client.classify_page` is skipped entirely, saving execution time and API costs.
4. **Pass 2 Lookahead Integrity**: Write test cases for `_group_pages_into_documents` to guarantee that dummy categories do not break merging, and that isolated missing footers (e.g., page 3 is blank, but page 4 has a footer) are correctly absorbed into the group.
