# Phase 07.3: Improve multi-page correspondence processing via Arabic footer pattern detection - Context

**Gathered:** 2026-06-24
**Status:** Ready for planning

<domain>
## Phase Boundary

Drastically speed up processing by detecting multi-page correspondences and skipping page-by-page LLM extraction for the subsequent pages of the same document.

</domain>

<decisions>
## Implementation Decisions

### Pattern Matching Strictness
- **D-01:** Flexible OCR matching — Allow minor whitespace/character variations due to OCR imperfections when detecting Arabic footers like "1 من 10 الصفحة".

### Grouping Assurance
- **D-02:** Lookahead window — If page N is missing the footer entirely but page N+1 has the footer, tolerate the miss and include page N in the group (as long as sequence eventually continues). A complete stop/absence of footers breaks the sequence and marks a new document.

### LLM Skip Mechanism
- **D-03:** Use Mac's Native Vision OCR (`pyobjc-framework-Vision`) on the bottom 10% of the page to extract footer text. It's built-in, fast, handles Arabic well, and avoids running the heavy LLM call on skipped pages.

### the agent's Discretion
None
