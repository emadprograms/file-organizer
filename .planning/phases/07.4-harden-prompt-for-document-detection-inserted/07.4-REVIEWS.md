# Cross-AI Plan Review — Phase 07.4

## Antigravity Review (Self)

**Summary:**
The plan thoroughly covers the required prompt hardening. It introduces explicit Arabic subject patterns and conditional logic for when a subject is missing. It correctly incorporates recent critical user constraints regarding the strict definition of `amar_takhsees` and the `30 bd / 60 bd` disambiguation between rent and allowance deductions. The schema update ensures robust signaling for the fallback mechanism without breaking existing typed responses.

**Strengths:**
- **Robust Schema:** Using `needs_gemma_fallback: bool` in the Pydantic schema is cleaner than parsing unstructured text or error handling.
- **Strict Disambiguation:** Explicitly calling out the presence of "30 bd" or "60 bd" for rent deduction prevents the AI from mixing up the two visually similar Arabic phrases.
- **Penalty on False Positives:** The instructions strictly forbid classifying random documents as `amar_takhsees`, anchoring the decision to a "higher authority" order.

**Concerns:**
- **LOW:** The fallback to `gemini-4-26b` might introduce a slight latency spike for documents with no subject. However, this is an acceptable tradeoff for accuracy.

**Suggestions:**
- When updating `classify_page()`, ensure the fallback logic logs the exact reason (e.g., `needs_gemma_fallback` was true) for observability.

**Risk Assessment:**
**LOW**. The changes are primarily prompt engineering and inference routing bounded by strong schema validation.
