# Phase 07.6: Semantic Rolling Window Grouping

## Goal
Replace the fragmented metadata-based grouping logic in Pass 2 with a semantic, "story-based" approach using a rolling window comparison of page summaries.

## Artifacts this phase produces
- `SemanticMatchResult` (class in `src/schemas.py`)
- `check_semantic_continuation` (function in `src/llm.py`)
- `tests/test_semantic_grouping.py` (new file)

## Tasks

<task>

### 3. Integrate Semantic Grouping in Pipeline
<task>
<read_first>
- src/pipeline.py
- src/llm.py
</read_first>
<action>
Modify `src/pipeline.py`:
1. In `process_pdf`, delete the `extracted_footer` Vision framework OCR code and any references to `is_continuation` passed to `classify_page` or `extract_page`.
2. Update `_group_pages_into_documents` signature and logic:
   - For each page, evaluate its primary tenant as before.
   - Maintain a list of summaries for the current `DocumentGroup`.
   - Call `self.llm.check_semantic_continuation(current_group_summaries, page.summary)` to decide if the page belongs to the current group.
   - If true, extend the current `DocumentGroup`'s `end_page` and append the summary.
   - If false, append the completed group to `documents` and start a new `DocumentGroup`.
   - Log the LLM's `reason` using the logger.
</action>
<acceptance_criteria>
- `grep "import Vision" src/pipeline.py` returns no results.
- `_group_pages_into_documents` calls `self.llm.check_semantic_continuation`.
- The log statement for the semantic matching reasoning exists in `_group_pages_into_documents`.
</acceptance_criteria>
</task>
