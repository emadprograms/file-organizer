# Phase 07.6: Semantic Rolling Window Grouping - Context

**Gathered:** 2026-06-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Replacing the fragmented metadata-based grouping logic with a semantic "story-based" approach using a rolling window comparison of page summaries.
</domain>

<decisions>
## Implementation Decisions

### Model & Comparison Cost
- **D-01:** We will use a local LLM (specifically Qwen2.5 14b). The cost and latency for doing an LLM comparison per adjacent page pair is acceptable locally.

### Interruptions & Blank Pages
- **D-02:** A blank or mostly illegible page in the middle of a document must not break the group. Its summary should explicitly state something like "contents are empty and illegible" and the rolling window should carry over the group.

### Summary Strictness & Content
- **D-03:** Do not strictly force a rigid schema/format for the summary. Prefer the "This is for [Person] regarding [Topic]" structure, but allow the LLM to just "call it what it is" if it encounters weird pages (like blanks). 
- **D-04:** The topic description within the summary should be verbose and clearly explain exactly what is going on in the page/letter/form, not just a single word.

### Independent Processing Edge Cases
- **D-05:** Pages that only contain a signature and nothing else are essentially junk ("BS"). They should be summarized as such and should not disrupt the document group.

</decisions>
