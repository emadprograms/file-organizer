# Phase 07.6: Semantic Grouping & Data Cleaning - Context

**Gathered:** 2026-06-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Data Cleaning & Interpolation layer in Pass 1.5, and Bulk Semantic Grouping in Pass 2. This completely replaces the fragmented metadata-based grouping logic, the old entity-resolution identity erasure, and the slow rolling-window concept.
</domain>

<decisions>
## Implementation Decisions

### 1. Pass 1: Hijri Date Ban & Summary Quality
- **D-01:** The `_build_system_prompt()` for Pass 1 must explicitly ban extracting Hijri dates (e.g., 1445 AH). Only extract Gregorian dates. If both are present, ignore the Hijri date.
- **D-02:** The `summary` instruction in Pass 1 must force a solid, verbose "full story" of the page in one line. Tell the AI not to be lazy with the summary.

### 2. Pass 1.5: Date Cleaning & Interpolation (Global Pass)
- **D-03:** Implement a global Python pass (no LLM) that reads all extracted dates from Page 1 to N, determines the overall chronological trend, spots empty (`None`) or obviously out-of-sequence dates, and interpolates them based on the nearest valid dates. No sliding window needed.

### 3. Pass 1.5: Dependent Alias Mapping (Names)
- **D-04:** Instead of the LLM guessing family hierarchies, Pass 1.5 will look for names extracted from `personal_details` category pages. Any name found on these pages is mapped directly as an alias to the active Primary Tenant. (Note: A primary tenant is established by an Anchor document + >3 mentions of the name). 
- **D-05:** All future documents addressed to these alias names will be routed to the Primary Tenant's timeline.

### 4. Pass 2: Bulk Semantic Grouping
- **D-06:** Replace the 1-by-1 rolling window with a single bulk LLM call. Pass a JSON array of all pages (page number, names, summaries) to the local LLM and ask it to return grouped page ranges (e.g., `[[1, 2, 3], [4], [5, 6]]`). 
- **D-07:** If the document is massive, chunk the array into 50-page batches to prevent context window / attention loss.

</decisions>
