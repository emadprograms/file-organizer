# Phase 23 - Plan 01: Inbox Syntax Parser

## Requirements
- FSUI-01

## Goal
Create the pure syntax parser that extracts the 5 positional arguments and title from a space-separated filename, mapping them to a Pydantic model.

## must_haves
- ParsedCommand model supports 5 positional fields and a title
- Invalid filenames throw specific exceptions to allow renaming
- No filesystem or LLM side-effects in this pure parsing step

  <action>
    Create src/inbox/parser.py and implement parse_filename_syntax function that takes a filename string and returns a ParsedCommand.
    Logic:
    - If filename.lower().endswith(".pdf"), securely strip the extension using filename[:-4].
    - Split by space character.
    - If the resulting list has fewer than 5 elements, raise a ValueError with message "Invalid Format".
    - Map the first 5 elements to area, house, tenant_hint, group, and date respectively.
    - Join any remaining elements with a space and assign to title.
    - Return the populated ParsedCommand.
  </action>
  <acceptance_criteria>
    - python -c "from src.inbox.parser import parse_filename_syntax; cmd = parse_filename_syntax('SAF 1234 Ali 1 2026-01-01 My Title.pdf'); assert cmd.area == 'SAF'; assert cmd.title == 'My Title'"
    - Function raises ValueError when passed 'SAF 1234.pdf'
    - Filename extension stripping uses slicing ([:-4]) instead of .strip()
  </acceptance_criteria>
</task>
