<feature>
  <name>Pipeline Decoupling and Routing Checkpoints</name>
  <files>src/processing/pipeline.py</files>
  <behavior>
    - Ensure routing is a distinct loop executed after all grouping is complete.
    - Routing loop iterates over `documents`. For each document, if its index is in `processed_indices`, skip. Otherwise, route it, update `processed_indices`, and `save_state()`.
    - Sanity check: Before resuming routing, compute a checksum (e.g., hash of all group start/end pages and categories) and compare it with the stored `grouping_checksum` in `RoutingState`. If they don't match, log a warning and start routing from scratch (clear state).
  </behavior>
  <implementation>
    1. In `src/processing/pipeline.py`, update `_group_and_route_documents`.
    2. Add logic to compute a `grouping_checksum` from the `documents` list (e.g., `hashlib.sha256` of concatenated `start_page`, `end_page`, `category`).
    3. Initialize `RoutingStateManager` using a new checkpoint path (e.g., `run_checkpoint_path.replace('.json', '_routing.json')` if `run_checkpoint_path` is present).
    4. Load state. If `state.grouping_checksum` exists and doesn't match the current checksum, log and reset state.
    5. Iterate `enumerate(documents)`. Skip if `i in state.processed_indices`.
    6. Route the document, passing any model config if available (for now, use default).
    7. Add `i` to `state.processed_indices`, update `state.grouping_checksum`, and save state.
    8. Once complete, clean up the routing checkpoint file.
  </implementation>
</feature>
