# Phase 07.5.2: Pass 1a Cloud-First Vision Extraction with Local Fallback - Context

**Gathered:** 2026-06-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Refactor the Pass 1a/1b pipeline architecture. Cloud requests must perform direct classification (`image -> PageClassification`) on `gemma-4-26b-a4b-it`, respecting a strict 15 RPM global IP-level rate limit. When rate limits or cooldowns are active, the system must instantly fall back to local OCR (`qwen2.5vl:7b`) to extract raw Arabic text (Pass 1a) without waiting or sleeping. The local text classification (Pass 1b via `qwen2.5:14b`) is deferred to the end of the file to prevent page-by-page model-swapping and maintain maximum processing speed.
</domain>

<decisions>
## Implementation Decisions

- **D-01: Direct Cloud Path:** Cloud calls perform direct vision-to-classification (`Image -> PageClassification`) via `gemma-4-26b-a4b-it`. We do not do separate OCR + text classification when using the cloud.
- **D-02: Strict Global IP Cap:** Enforce a global cap of 15 RPM across the 44 keys in the sliding window. Cycle keys purely for daily/monthly total quota, not to beat the 15 RPM IP limit.
- **D-03: Sustained Local Fallback (OCR only):** On hitting the 15 RPM limit or receiving a 429, set `GemmaClient.global_cooldown_until = now + 65.0`. During this cooldown, instantly fall back to local OCR (`qwen2.5vl:7b`) for all incoming pages. Do not sleep the thread.
- **D-04: Deferred Reasoning (Pass 1b):** Store all raw text extracted locally in a deferred list. At the end of the PDF processing, load/use the local reasoning model (`qwen2.5:14b`) to classify these deferred pages.
</decisions>
