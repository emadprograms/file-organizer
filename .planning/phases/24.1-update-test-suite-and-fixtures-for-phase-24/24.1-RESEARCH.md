# Phase 24.1 Research: Update test suite and fixtures for Phase 24

## 1. Summary of Existing Test Structure

The existing E2E test suite uses mocked, isolated fixtures and pre-saved state files to verify the file-organizer pipeline deterministically.

**Key Files:**
*   **`tests/test_pipeline_e2e.py`**: Tests `create` and `append` dry runs/full runs using `tests/fixtures/golden_1273/` test structures. Relies on `_setup_run_dir()` to copy an isolated, minimized 1273 mock house.
*   **`tests/test_pipeline_e2e_continuity.py`**: Tests LLM chunk grouping continuity logic using `tests/fixtures/unit_states/continuity_test_state.json`. **This is unaffected by Phase 24** (Decision D-03).
*   **`tests/test_pipeline_e2e_default.py` & `test_pipeline_e2e_others.py`**: Unit-like E2E tests focusing on grouping logic and precision, reading from the old `pdfs/1273/567_report.json` path.

**Existing Assertions:**
The tests assert whether strings like `"1273"` or `"يونس"` appear in standard output, and they perform simple checks on whether `output_dir / "1273 - يونس محمد مالك"` is created.

## 2. Inventory of New Fixtures

The new fixtures represent a much more authentic structure, reflecting actual complex histories.

**Areas Structure (`tests/fixtures/e2e/golden_state/areas/`):**
*   **`Safra C / 1273 - يونس محمد ملاك/`**: A house with a single tenant since 2002. Contains a finalized PDF (`1273_finalized.pdf`) but **is missing** the `.source_files` directory (and thus missing `1273_tenants.yaml`).
*   **`Safra D / 504 - أحمد يوسف المريسل/`**: A complex house with multiple tenants over time (2001-2011, 2012-2019, 2021-Present, plus an unassigned period). Includes a complete `.source_files/` directory, full folder structure, and a `504_finalized.pdf`.

**Inbox PDFs (`tests/fixtures/e2e/golden_state/inbox/`):**
1.  `SAFC 1273 U G U.pdf` (Area=SAFC, House=1273, Group=General "G", missing tenant and date)
2.  `SAFC 1273 يونس 7 U.pdf` (Area=SAFC, House=1273, Tenant=يونس, Group=7, missing date)
3.  `U 1273 U U U.pdf` (Only House is explicit)
4.  `U 504 U U U.pdf` (Only House is explicit)
5.  `broken.pdf` (No parsable syntax)

## 4. Pattern Analysis for LLM Mock Files (`_report.json`)

Currently, tests mock `src.grouping.process_with_shrink`, `src.routing.route_document`, and `src.timeline.phase.canonicalize_with_llm` heavily.
Going forward, to test the true orchestration:
*   We need to pre-generate `_report.json` for each of the 5 new Inbox PDFs.
*   These reports will act as the mocked response from Gemini 3.1 Flash Lite.
*   We will place these pre-saved reports alongside the new fixtures, and our tests will inject them (or place them in the working `.source_files/` directory) to avoid real API calls while preserving realistic grouping/routing data.

## 5. Technical Approach for Updating Assertions

Following **Decision D-01 (Hardcode expectations)**:
1.  Instead of just checking standard output or high-level directory existence, we will build a static dictionary mapping each Inbox PDF to its exact expected final state.
2.  **Example Assertion:** After processing `SAFC 1273 يونس 7 U.pdf`, we will assert `assert (house_dir / "يونس محمد ملاك ‎(2002 - الآن)‎" / "07_استقطاع إيجار" / "expected_new_filename.pdf").exists()`.
3.  **Isolation:** For each test, we will use `shutil.copytree()` to copy the pristine `golden_state/areas/Safra D/504...` (or 1273) to `tmp_path`, run the `append` or `create` CLI command against it, and verify the resulting folder structure inside `tmp_path`.

## 6. Risks and Edge Cases

*   **Missing YAML for 1273:** The 1273 golden state is missing its `.source_files` directory and the `_tenants.yaml` file. If a test tries to append to 1273, Requirement **FSUI-06** will trigger, causing the system to abort and rename the file to `_Error_Missing_YAML.pdf`. This is actually a *good* thing, as it naturally covers the FSUI-06 edge case, but we must ensure we intentionally test it rather than failing unexpectedly.
*   **Deterministic Inference:** For tests inferring data (e.g., `U 504 U U U.pdf`), the mock `_report.json` must be carefully crafted to ensure the pipeline correctly identifies the house and tenant based on the mock OCR content.
*   **Broken Syntax Fallbacks:** `broken.pdf` may throw exceptions depending on how the CLI/Inbox parser handles completely invalid filenames. We need to ensure the test catches the specific expected failure (e.g., skipped file, logged error, or renaming to `_Error...`).
