# Objective
Generate deterministic mock `_report.json` files for the new Inbox PDFs to enable completely isolated, offline testing of the pipeline without hitting the live Gemini API.

# Must Haves
- The reports must contain valid `category`, `expected_tenant_name`, `date`, `content_explanation`, and `subject` fields that naturally map to the real houses (Safra C/1273 and Safra D/504).
- The reports must cover all 4 valid test PDFs (broken.pdf does not need a report).
