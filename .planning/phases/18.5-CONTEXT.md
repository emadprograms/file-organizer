# Phase 18.5 Context: Finalize PDF Output, Compression, and Metadata

## Overview
This phase ensures that the original categorized PDF is finalized gracefully. Instead of just producing individual routed segment files, the pipeline will compress the entire combined PDF, add a Table of Contents (metadata bookmarks) mapping each page to its assigned `Tenant - Folder Name`, and save it as `_finalized.pdf`.

## Gray Areas Resolved

### 1. "Metadata" mapping to folders
**Decision:** We will inject a Table of Contents (PDF Bookmarks/Outlines) into the final combined PDF. 
- **Format:** Each bookmark will point to the respective page and be titled `Tenant - Folder Name` (e.g., `1273 - some_folder_name`). 
- **Reasoning:** Since metadata fields like `Title` or `Subject` apply to the entire document, a Table of Contents is the only way to "metadata-tag" each individual page within a single combined PDF to show where it was routed.

### 2. PDF Compression
**Decision:** We will reuse our existing `compress_pdf` logic (located in `src/pdf/compress.py`), but apply it to the main combined PDF.
- **Details:** Currently, `compress_pdf` is only called on the individual extracted page files. We will now apply this compression to the full `_categorized.pdf` before (or during) its conversion to `_finalized.pdf`. We will ensure the quality stays at a "normal" level (the current logic scales images > 1500px and uses `garbage=4, deflate=True`).

### 3. File Naming
**Decision:** The original `[house_id]_categorized.pdf` (e.g., `1273_categorized.pdf`) will be processed and saved as `[house_id]_finalized.pdf`. This finalized file will be placed in the root of the output directory alongside the tenant folders.

## Implementation Guidelines for Planners
- Locate where `src/main.py` processes the target directory.
- Update `run_generation_pass` (or add a step after it) to process the main PDF:
  1. Add TOC/bookmarks based on the `per_page` routing logic mapping.
  2. Compress the document using `src.pdf.compress.compress_pdf`.
  3. Save the result as `[house_id]_finalized.pdf`.
- Verify that `extract_pdf_segment` doesn't needlessly re-compress segments if the main PDF is already compressed, optimizing performance.
