# Phase 07.5.1: Hybrid Cloud-First Vision Extraction with Local Overflow - Context

**Gathered:** 2026-06-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Implement a hybrid cloud/local approach for Pass 1a (Vision Extraction) to bypass the 15 requests per minute rate limit of the cloud model. It uses the cloud model first, and during its 1-minute cooldown period, seamlessly falls back to a local model for vision extraction, reverting to the cloud model once it becomes available again.
</domain>

<decisions>
## Implementation Decisions

### Vision Model Architecture (Pass 1a only)
- **D-01:** **Primary Model:** Use the cloud model for Pass 1a until rate limited (15 tries/min).
- **D-02:** **Fallback Model:** When the cloud model is rate limited, use the local model for vision extraction during the 1-minute cooldown period.
- **D-03:** **Resumption:** Automatically switch back to the cloud model as soon as the rate limit window resets.
- **D-04:** **Scope:** This hybrid approach is strictly limited to Pass 1a (Vision text extraction).

### the agent's Discretion
- State management and exact logic to track the 15-request limit and 1-minute cooldown timer.
- Error handling and retry logic during the handoff between cloud and local models.

</decisions>
