# Phase 07.5.1: Hybrid Cloud-First Vision Extraction with Local Overflow - Research

## Overview

The purpose of this phase is to mitigate the processing bottlenecks caused by the cloud vision model's strict rate limits (15 requests per minute) during Pass 1a. By implementing a hybrid approach, the system will seamlessly failover to a local vision model when the cloud model enters its rate-limit cooldown, and revert to the cloud model once the cooldown expires.

## Analysis of Requirements

1. **Scope:** Limited strictly to Pass 1a (Vision Extraction). Pass 1b and later passes are not affected by this logic.
2. **Primary Model:** Cloud model (e.g., Gemini). Fast and highly capable, but constrained to 15 rpm.
3. **Fallback Model:** Local model (e.g., Qwen2.5-VL-7B-Instruct via LM Studio). Used as an "overflow" processing queue during the 1-minute cloud cooldown.
4. **State Management:** The system needs robust tracking of:
   - The current number of cloud requests in the sliding 1-minute window.
   - A cooldown timer or timestamp indicating when the cloud model will be available again.
5. **Seamless Handoff:** Error handling must catch `429 Too Many Requests` (or internal tracking hitting 15), immediately redirect the current request to the local model, and flag the system as in "cooldown" for 60 seconds.

## Architectural Strategy

### 1. State Tracking Mechanism
Introduce a state manager (e.g., `VisionModelManager` or an extension of the existing telemetry/API manager) that maintains:
- `cloud_request_timestamps`: A list of timestamps for the last 15 cloud requests.
- `cloud_cooldown_until`: A timestamp tracking the expiration of the 1-minute penalty.

### 2. Execution Flow
When a page requires Pass 1a extraction:
1. **Check Cooldown:** 
   - If `current_time < cloud_cooldown_until`, bypass cloud and route immediately to local model.
2. **Check Capacity:**
   - Clean up `cloud_request_timestamps` older than 60 seconds.
   - If `len(cloud_request_timestamps) < 15`, attempt cloud extraction.
3. **Cloud Extraction Attempt:**
   - Execute the cloud API call.
   - On Success: Record the timestamp. Return the result.
   - On 429 Rate Limit (or other exhaustion error): 
     - Set `cloud_cooldown_until = current_time + 60 seconds`.
     - Fallback to the local model to process the current request.
4. **Local Extraction:**
   - Execute the local API call (LM Studio).
   - Return the result.
   
### 3. Integration Points
- **Pass 1a Module:** The core processing loop needs to be updated to delegate the actual inference call to our new hybrid logic, rather than hardcoding the cloud call.
- **Telemetry/GUI:** Updates might be necessary to visually indicate when the system is in "Local Overflow" mode vs "Cloud" mode.

## Validation Architecture

To ensure the hybrid approach works correctly and safely, we will employ the following validation strategy:

1. **State Management Unit Tests:**
   - Verify that recording 15 requests within 60 seconds correctly triggers the capacity limit.
   - Verify that timestamps older than 60 seconds are purged, freeing up capacity.
   - Verify that receiving a simulated 429 correctly sets the cooldown timer for 60 seconds.

2. **Execution Flow Validation:**
   - Mock both cloud and local model clients.
   - **Test Normal Flow:** Ensure requests < 15/min hit the cloud mock and NOT the local mock.
   - **Test Overflow Flow:** Force a 429 from the cloud mock on the 16th request. Verify the request is caught, the cooldown is set, and the result is returned from the local mock.
   - **Test Cooldown Flow:** While in cooldown, verify new requests immediately hit the local mock without attempting the cloud mock.
   - **Test Resumption Flow:** Advance the system clock by >60 seconds. Verify the next request correctly targets the cloud mock again.

3. **End-to-End & Telemetry Verification:**
   - Process a sample dossier with >20 pages.
   - Monitor the application logs and GUI telemetry to ensure the system processes the first ~15 pages via cloud, cleanly transitions to local processing for the remaining pages within that minute, and continues without pausing or crashing.
