<feature>
  <name>Resilient Grouping Loop</name>
  <files>src/processing/grouping/core.py, tests/test_grouping.py</files>
  <behavior>
    - State Integration: Load initial state from `GroupingStateManager`; save state after every LLM interaction.
    - Adaptive Shrinking: 
        - Success: Reset `chunk_size_index` to 0 (size 4).
        - Rotation Failure: Increment `chunk_size_index` (4 -> 3 -> 2) only upon catching `ProviderRotationExhaustedError` from `LLMClient` 3 consecutive times.
    - Graceful Halt: If `chunk_size_index` is at 2 and a rotation failure occurs, call `save_state()` to persist final state immediately before raising `GracefulHaltException`.
    - Anchor Merge: Use the refactored `merge_chunks` for overlapping pages.
  </behavior>
  <implementation>
    1. Refactor `process_with_shrink` in `src/processing/grouping/core.py`.
    2. Replace local counters with `GroupingStateManager` calls.
    3. Update `CHUNK_SIZES` constant to `[4, 3, 2]`.
    4. Wrap LLM calls in a try-except block that specifically catches `ProviderRotationExhaustedError` (from `LLMClient`) and increments a consecutive failure counter.
    5. Implement the reset-on-success and shrink-on-rotation-failure logic.
    6. Add TDD tests in `tests/test_grouping.py` simulating:
       - Successful process -> size stays 4.
       - 3 Rotation failures -> size shrinks 4->3.
       - Rotation failure at size 2 -> Graceful Halt (with state saved prior to exception).
       - Resume from state file after halt.
       - Partial Success (e.g. Provider A fails, Provider B succeeds) -> size stays 4 and DOES NOT shrink.
  </implementation>
</feature>
