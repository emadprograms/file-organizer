<feature>
  <name>Anchor Page Merging</name>
  <files>src/processing/grouping/utils.py, tests/test_grouping.py</files>
  <behavior>
    - Overlap Identification: Given two chunks and an `overlap_page_idx`, identify the groups containing that page in both chunks.
    - Boundary Validation: Merge the chunks only if the anchor page is part of a logical document group (not a fragment) in both. 
    - Conflict Resolution: If LLM grouping decisions conflict on the Anchor Page (e.g. one chunk says start of doc, other says end), default to splitting the document at the chunk boundary.
    - Attribute Preservation: Preserve the tenant/category metadata from the primary source of truth (Chunk N).
  </behavior>
  <implementation>
    1. Refactor `merge_chunks` in `src/processing/grouping/utils.py` to explicitly use the `overlap_page_idx` as the anchor.
    2. Implement logic to find the `DocumentGroup` containing the anchor page in both the previous and current result set.
    3. Only merge if the anchor page exists in both and is part of a continuing document.
    4. Add TDD tests in `tests/test_grouping.py` with three scenarios:
       - Scenario A: Anchor page is a continuation (MERGE).
       - Scenario B: Anchor page is the end of Doc 1 and start of Doc 2 (SPLIT).
       - Scenario C: Anchor page is a fragment (SPLIT).
       - Scenario D: Conflict on Anchor Page decisions -> fallback to SPLIT.
  </implementation>
</feature>
