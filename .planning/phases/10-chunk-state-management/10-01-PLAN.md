<feature>
  <name>Grouping State Manager</name>
  <files>src/processing/grouping/state.py, tests/test_grouping.py</files>
  <behavior>
    - State Schema: Must track `current_page_index` (int), `chunk_size_index` (int), `failure_count` (int), and `processed_groups` (list).
    - Atomic Save: `save_state()` writes to a `.tmp` file then renames to avoid corruption during crashes. Should also maintain a `.state.json.bak` fallback.
    - Load State: `load_state()` returns a state object or defaults if no file exists. Fallback to `.state.json.bak` if main state file is corrupted/0-bytes.
    - Validation: Use Pydantic to ensure the loaded JSON matches the expected schema.
  </behavior>
  <implementation>
    1. Define `GroupingState` Pydantic model in `src/processing/grouping/state.py`.
    2. Implement `GroupingStateManager` class with `load_state` and `save_state(state: GroupingState)` methods.
    3. Use `os.replace` for atomic writes and maintain a `.bak` copy of the previous valid state.
    4. Implement tests in `tests/test_grouping.py` verifying:
       - State is correctly saved and loaded.
       - Malformed JSON is handled gracefully.
       - Atomic write preserves data on interrupted write (mocked).
  </implementation>
</feature>
