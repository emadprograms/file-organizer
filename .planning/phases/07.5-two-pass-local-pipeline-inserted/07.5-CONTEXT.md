## Phase Boundary

Completely eliminate local vision model reasoning failures by decoupling OCR and categorization. Split the current Pass 1 into two parts: Pass 1a (Qwen2.5-VL for Arabic text extraction) and Pass 1b/1.1 (Local text model like llama3.1:8b for 13-category JSON classification).

</domain>

<decisions>
## Implementation Decisions

### Reasoning Model Selection
- **D-01:** Use a local text model (e.g. llama3.1:8b or qwen2.5:7b) via local API for the Pass 1b (1.1) classification step. This keeps the pipeline 100% local, avoiding external dependencies while using small specialized models for reasoning.

### Failure and Retry Strategy
- **D-02:** If Pass 1b (Local text model) fails to return the required JSON schema, implement a tiered retry logic:
  1. First, retry the local text model call using the already extracted OCR text.
  2. If that fails again, re-run Pass 1a (Qwen) to get a fresh OCR extraction, and then attempt the text model classification again.

### Vision Output Format
- **D-03:** Prompt Qwen2.5-VL in Pass 1a to explicitly maintain the spatial layout and line breaks of the original document. This spatial context is critical for helping the text model understand the document structure during classification.

### the agent's Discretion
- Code structure for retry logic (loops vs decorators).
- Exact local text model choice (e.g., Llama 3.1 8B vs Qwen 2.5 7B), though a standard chat model is likely sufficient for JSON extraction.
</decisions>
