<domain>
## Phase Boundary

Completely eliminate local vision model reasoning failures by decoupling OCR and categorization. Split the current Pass 1 into two parts: Pass 1a (Qwen2.5-VL for Arabic text extraction) and Pass 1b/1.1 (DeepSeek API for 13-category JSON classification).

</domain>

<decisions>
## Implementation Decisions

### Reasoning Model Selection
- **D-01:** Use the DeepSeek API for the Pass 1b (1.1) classification step. This provides high-speed, cheap, and accurate JSON reasoning on the extracted text.

### Failure and Retry Strategy
- **D-02:** If Pass 1b (DeepSeek) fails to return the required JSON schema, implement a tiered retry logic:
  1. First, retry the DeepSeek API call using the already extracted OCR text.
  2. If that fails again, re-run Pass 1a (Qwen) to get a fresh OCR extraction, and then attempt the DeepSeek classification again.

### Vision Output Format
- **D-03:** Prompt Qwen2.5-VL in Pass 1a to explicitly maintain the spatial layout and line breaks of the original document. This spatial context is critical for helping DeepSeek understand the document structure during classification.

### the agent's Discretion
- Code structure for retry logic (loops vs decorators).
- Exact DeepSeek model endpoint (e.g., `deepseek-chat` or `deepseek-reasoner`), though a standard chat model is likely sufficient for JSON extraction.
</decisions>
