# Phase 07.5: Two-Pass Local Pipeline

## Context & Goal
Completely eliminate local vision model reasoning failures by decoupling OCR and categorization. Split the current Pass 1 into two parts: Pass 1a (Qwen2.5-VL for Arabic text extraction) and Pass 1b/1.1 (Local text model like llama3.1:8b for 13-category JSON classification). Implement a tiered retry logic for the text model and Qwen, while preserving the existing Gemini 4 26b fallback for edge cases.

## Tasks

<action>
Refactor `GemmaClient.classify_page` in `src/llm.py` to implement the two-pass logic.

Create two new helper methods on `GemmaClient`:
- `_extract_text_with_qwen(self, image_bytes: bytes) -> str`: Calls `qwen2.5vl:7b` via `self.local_client.chat.completions.create` (not `.parse`). Pass a prompt instructing it to "Transcribe all Arabic text from the image verbatim, explicitly maintaining the spatial layout and line breaks. Do not attempt to classify or summarize."
- `_classify_text_with_local_model(self, text: str) -> PageClassification`: Sends the transcribed text to a local text model (e.g., `llama3.1:8b`) via `self.local_client.chat.completions.parse` along with the existing `self._build_system_prompt()`. Request structured output using the `PageClassification` schema and return it.

Update `classify_page` to orchestrate them with tiered retries:
1. Implement an inner loop: retry the text classification up to 3 times if it fails to return valid JSON or encounters an API error.
2. Implement an outer loop: if the inner loop fails all 3 times, re-run Qwen for a fresh OCR extraction and try the text classification again (allow up to 2 Qwen extractions).
3. If the local text model successfully returns a result but sets `needs_gemma_fallback = True`, raise a `ValueError("Local Inference Refused via needs_gemma_fallback flag")` to trigger the existing fallback.
4. Ensure the `gemini-4-26b` fallback block catches any failures from the two-pass pipeline (including exhausted retries or the fallback flag) and calls `self._route_llm_call(model='gemini-4-26b', ...)`.
</action>
