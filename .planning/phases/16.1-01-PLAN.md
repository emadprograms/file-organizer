# Plan: 16.1 Cleanup Checkpoints System

## Objective
Refactor the checkpointing and intermediate state saving logic across the pipeline to use a unified `.run_cache/` folder. Rename intermediate files to clearly track their pass, and implement a feature to preserve these files into the final `source_files/` directory instead of deleting them. This provides granular resume/redo capabilities if the pipeline fails or needs a specific pass re-run.

## Context
See `.planning/phases/16.1-CONTEXT.md` for the explicit decisions.
- Pass 1 state: `{house_id}_1_cleaned.json`
- Pass 2 state: `{house_id}_2_grouped.json` (consolidates grouping state)
- Pass 3 state: `{house_id}_3_routed_and_finalized.json` (replaces manifest.json)
- Checkpoint directory: `.run_cache/`
- All JSON checkpoints moved to `{house_id}/source_files/` upon successful completion.

## Step 1: Update src/main.py Checkpoint Paths
- Update `run_cleaning_pass` to save and look for `{house_id}_1_cleaned.json` inside the `.run_cache/` directory.
- Update `run_grouping_pass` to use `.run_cache/` instead of `checkpoints/`. Set both the grouped checkpoint path and the run checkpoint path to point to `.run_cache/{house_id}_2_grouped.json`.
- When passing the checkpoint path into `pipeline._group_and_route_documents()`, ensure it points to the single `.run_cache/{house_id}_2_grouped.json`.

## Step 2: Update src/pipeline/pipeline.py Cleanup Logic
- Update `_route_documents` to **remove** the logic that deletes checkpoints on successful completion. We now want to preserve them.
- Ensure any `_routing.json` suffix or `_chunk_` suffixes still work but are not deleted prematurely, or ideally just leave cleanup to `main.py` completely.

## Step 3: Update src/timeline/reconciliation.py
- Change the `manifest_path` logic. It should write the manifest to `output_dir / ".run_cache" / f"{house_id}_3_routed_and_finalized.json"` instead of `output_dir / f"{house_id}_manifest.json"`.

## Step 4: Update src/main.py Generation & Final Cleanup Logic
- In `run_generation_pass`:
  - Locate `1_cleaned.json`, `2_grouped.json`, and `3_routed_and_finalized.json` inside `.run_cache/`.
  - Check if they exist. If so, `shutil.move()` them into the `source_files_dir`.
  - Remove the old logic that deleted the `checkpoints/` directory and `cleaned.json`.
  - Remove the old logic that moved `manifest_path` (since it is now handled as `3_routed_and_finalized.json`).

## Step 5: Verify Changes
- Run the tests to ensure `main.py` and pipeline functions still behave correctly.
- Do a dry-run to see that the `.run_cache/` folder is correctly utilized and files are properly renamed and moved to `source_files/`.
