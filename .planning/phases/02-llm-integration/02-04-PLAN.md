# Plan 02-04: Fix Invalid Argument Error in Gemma Client

## Objective
Remove the unsupported `thinking_config` parameter from the Gemma 4 31b-it client configuration to prevent the `400 INVALID_ARGUMENT` crash on startup.

## Requirements Addressed
- Gap closure for UAT Test 1 (Pipeline crash on startup)

## Implementation Steps

### 1. Update Gemma Client Configuration
- **File:** `src/llm.py`
- **Action:** In the `classify_page` method, locate the `types.GenerateContentConfig` instantiation.
- **Change:** Remove the `thinking_config=types.ThinkingConfig(thinking_budget=0),` parameter. The `gemma-4-31b-it` model does not support thinking budgets via the GenAI SDK, and omitting it will allow the model to run normally.

## Verification
- Run `python src/main.py sample.pdf` and ensure it no longer crashes on startup with `google.genai.errors.ClientError: 400 INVALID_ARGUMENT`.
- Run unit tests `python -m pytest tests/` to ensure the mock fixtures and logic remain unbroken.
