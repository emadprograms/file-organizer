# Plan 03 — Sequential Pipeline & Sliding Window Accumulator

## Goal

Replace the parallel `ThreadPoolExecutor` pipeline with a sequential page accumulator that implements the dynamic sliding window described in D-03. The pipeline feeds OCR-extracted text (not images) to `GemmaClient.classify_page()`, accumulates pages within a topic group, and emits completed document segments with their metadata.

## must_haves

- [ ] `PdfIngestor.extract_pages_as_text()` yields `(page_index, text_string)` tuples
- [ ] `PdfIngestor.extract_pages_as_text()` raises `NoTextLayerError` if the PDF lacks a text layer
- [ ] `Pipeline.process_pdf()` processes pages sequentially (no ThreadPoolExecutor)
- [ ] Sliding window enforces a `MAX_CONTEXT_PAGES` limit to prevent token exhaustion
- [ ] Sliding window accumulates pages until `is_continuation=False`
- [ ] `previous_summary` context is passed between groups
- [ ] Each emitted document contains `start_page`, `end_page`, `house_number`, `resident`, `category`
- [ ] Pipeline returns `list[DocumentGroup]` dataclass, not raw dicts
- [ ] Test stubs `test_continuation_detection` and `test_sliding_window` are unskipped and implemented

## must_haves.prohibitions
- [ ] Pipeline MUST NOT accumulate unbounded context pages (enforced by `MAX_CONTEXT_PAGES`)

## Artifacts this phase produces

### New dataclass in `src/schemas.py`
- `class DocumentGroup` — fields: `start_page: int`, `end_page: int`, `house_number: str`, `resident: str`, `category: Category`

### Refactored file: `src/ingest.py`
- `class NoTextLayerError(Exception)` — new exception for PDFs with no text
- `PdfIngestor.extract_pages_as_text(self, pdf_path: str) -> Iterator[Tuple[int, str]]` — uses PyMuPDF `page.get_text()` to extract text instead of rendering pixmaps, raising `NoTextLayerError` if the first page has no text
- `PdfIngestor.extract_pages_as_images()` — retained as-is for backward compatibility

### Refactored file: `src/pipeline.py`
- Removes `import concurrent.futures`
- `Pipeline.__init__(self, api_keys, delay_between_pages, max_context_pages)` — stores `delay_between_pages` and `max_context_pages: int = 10`, no `max_workers`
- `Pipeline.process_pdf(self, pdf_path: str) -> list[DocumentGroup]` — sequential loop:
  1. Iterates `ingestor.extract_pages_as_text(pdf_path)`
  2. Maintains `current_group_pages: list[str]`, `current_group_start: int`, `current_classification: PageClassification`, `previous_summary: str`
  3. Calls `client.classify_page(page_text, previous_summary, current_group_pages)`
  4. If `is_continuation=True` and `len(current_group_pages) < self.max_context_pages`: appends page to `current_group_pages`, updates `end_page`
  5. If `is_continuation=False` or `len(current_group_pages) >= self.max_context_pages`: emits `DocumentGroup(start_page, end_page, house_number, resident, category)`, resets accumulator, sets `previous_summary` to a brief string describing the completed group
  6. After loop ends, emits final group
  7. Returns `list[DocumentGroup]`

### Updated tests in `tests/test_llm.py`
- `test_continuation_detection()` — unskipped, tests that pipeline correctly groups continuation pages
- `test_sliding_window()` — unskipped, tests that accumulated_pages grows and resets on topic change
