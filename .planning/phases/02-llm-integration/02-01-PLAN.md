# Plan 01 — Schema Definition & LLM Client Refactor

## Goal

Replace the existing `GemmaClient.process_image()` method with a text-based `classify_page()` method that uses native structured JSON output via Pydantic schemas, targets the `gemma-4-31b-it` model, and implements robust retry logic with exponential backoff and key rotation.

## must_haves

- [ ] `Category` enum with exactly 13 members defined in `src/schemas.py`
- [ ] `PageClassification` Pydantic model with `house_number`, `resident`, `category`, `is_continuation` fields
- [ ] `GemmaClient.classify_page()` sends extracted text (not images) to `gemma-4-31b-it`
- [ ] Native JSON schema mode via `response_mime_type="application/json"` + `response_schema=PageClassification`
- [ ] `ThinkingConfig(thinking_budget=0)` disables thinking mode
- [ ] Retry decorator with exponential backoff (min=4s, max=60s) and max 7 attempts
- [ ] Key rotation on 429/rate-limit errors
- [ ] `pydantic` added to `requirements.txt`

## Artifacts this phase produces

### New file: `src/schemas.py`
- `class Category(str, Enum)` — 13 members: `BASIC_DETAILS`, `PERSONAL_DETAILS`, `AMAR_TAKHSEES`, `KEY_HANDOVER`, `CONTRACT`, `EWA_LETTERS`, `RENT_DEDUCTION`, `ALLOWANCE_DEDUCTION`, `NOTIFICATIONS`, `MAINTENANCE`, `PICTURES`, `MODIFICATIONS`, `OTHER_LETTERS`
- `class PageClassification(BaseModel)` — fields: `house_number: str`, `resident: str`, `category: Category`, `is_continuation: bool`

### Refactored file: `src/llm.py`
- `class InvalidResponseError(Exception)` — new exception for schema validation failures
- `class RateLimitError(Exception)` — retained
- `class GemmaClient.__init__(self, api_keys, delay_between_pages)` — adds `delay_between_pages: float = 1.0`
- `GemmaClient._build_system_prompt() -> str` — returns the classification system prompt with 13 categories + Arabic/English descriptions + special rules
- `GemmaClient.classify_page(self, page_text: str, previous_summary: str, accumulated_pages: list[str]) -> PageClassification` — replaces `process_image()`
- Retry decorator: `@retry(wait=wait_exponential(multiplier=2, min=4, max=60) + wait_random(0, 2), stop=stop_after_attempt(7), retry=retry_if_exception_type((RateLimitError, InvalidResponseError)))`
- `temperature=0` in `GenerateContentConfig`

### Modified file: `requirements.txt`
- Added line: `pydantic`

3. Update `GemmaClient.__init__` signature to `(self, api_keys: list[str] = None, delay_between_pages: float = 1.0)`. Store `self.delay_between_pages = delay_between_pages`.

4. Add method `_build_system_prompt(self) -> str` that returns the full classification system prompt containing:
   - Role: "Arabic document classification expert analyzing scanned housing files from Bahrain/Gulf region"
   - All 13 categories with bilingual descriptions (English identifier + Arabic name) exactly as listed in 02-RESEARCH.md lines 231–243
   - Special rules for NONE resident (D-08), name normalization (D-06), OCR noise tolerance
   - Placeholders for `{previous_summary}` and `{accumulated_pages}` in the prompt template

5. Remove `process_image()` method entirely.

6. Add new method `classify_page(self, page_text: str, previous_summary: str = "", accumulated_pages: list[str] = None) -> PageClassification`:
   - Builds the prompt using `_build_system_prompt()`, substituting `previous_summary` and the joined `accumulated_pages` text
   - Calls `self.client.models.generate_content()` with:
     - model="gemma-4-31b-it"
     - contents=[system_prompt_with_context]
     - config=types.GenerateContentConfig(response_mime_type="application/json", response_schema=PageClassification, thinking_config=types.ThinkingConfig(thinking_budget=0), temperature=0)
   - Returns `response.parsed` if not None
   - Falls back to `json.loads(response.text)` validated through `PageClassification(**data)` if `response.parsed` is None
   - Raises `InvalidResponseError` if both fail
   - On 429/rate-limit exceptions: calls `self.switch_key()` then raises `RateLimitError`
   - Calls `time.sleep(self.delay_between_pages)` after a successful call

7. Apply retry decorator to `classify_page`:
   `@retry(wait=wait_exponential(multiplier=2, min=4, max=60) + wait_random(0, 2), stop=stop_after_attempt(7), retry=retry_if_exception_type((RateLimitError, InvalidResponseError)))`
</action>
