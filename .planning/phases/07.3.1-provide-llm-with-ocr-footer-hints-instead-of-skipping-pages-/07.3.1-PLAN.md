# Phase 07.3.1: Provide LLM with OCR footer hints instead of skipping pages - Execution Plan 01

## Objective
Modify Pass 1 to execute the local LLM on every page, while passing the detected Arabic OCR pagination footer (e.g., "1 من 10") as a hint to the LLM prompt. This allows the LLM to make an informed decision on whether the page is a continuation, effectively eliminating the need for brittle lookahead heuristics in Pass 2.

## Tasks

### Task 01: Extract Arabic Footer from Vision OCR
**Description:** Modify Pass 1 in `src/pipeline.py` to extract the exact Arabic footer string instead of blindly setting a flag and skipping the LLM execution.
**Target File:** `src/pipeline.py`
**Autonomous:** true
**Actions:**
1. In the `process_single_page` method, locate the Vision OCR block (`request_handler.performRequests_error_`).
2. Remove the `footer_detected = True` logic that currently skips the LLM call and returns a mock `PageClassification`.
3. Inside the loop (`for observation in request.results():`), extract the matched footer text string (e.g., "1 من 10", "الصفحة 2") using the regex `r'(\d+\s*من\s*\d+|الصفحة\s*\d+|صفحة\s*\d+)'` and save it to a variable `extracted_footer`.
4. If an `extracted_footer` is found, log it (e.g., `print(f" Page {p_idx} has Arabic footer '{extracted_footer}'. Passing to LLM.")`).
5. Update the subsequent `classify_page` call to pass the new variable: `res = self.client.classify_page(image_bytes=i_bytes, footer_text=extracted_footer)`.

### Task 02: Update LLM Prompts and Method Signature
**Description:** Update the `classify_page` signature to accept the `footer_text` hint, conditionally append it to the user prompt, and clarify the schema definition.
**Target File:** `src/llm.py`
**Autonomous:** true
**Actions:**
1. Update the `classify_page` method signature: `def classify_page(self, image_bytes: bytes, footer_text: str = None) -> PageClassification:`.
2. Inside `classify_page`, conditionally append the hint to `user_prompt`: `if footer_text: user_prompt += f"\n\nHINT: The OCR system detected a page footer: '{footer_text}'. If this indicates it's the first page (like '1 من X'), it is NOT a continuation. If it indicates a subsequent page (like '2 من X' or 'الصفحة 2'), set is_continuation = true."`
3. In `_build_system_prompt()`, update the JSON instruction sentence at the bottom to explicitly request the boolean flags: `Return a JSON object with: house_number, residents (list of strings), category, date, is_continuation (boolean), is_form (boolean), and needs_gemma_fallback (boolean).`

### Task 03: Remove Lookahead Continuation Heuristics
**Description:** Since the LLM is now aware of the exact pagination strings, remove the artificial lookahead logic from the grouping loop in Pass 2.
**Target File:** `src/pipeline.py`
**Autonomous:** true
**Actions:**
1. In the `_group_pages_into_documents` method (or equivalent grouping logic), locate and delete the `lookahead_continuation` detection logic.
2. Ensure the grouping block strictly relies on the LLM's classification: `effective_continuation = getattr(page, 'is_continuation', False)`.

### Task 04: Integration & Manual Validation
**Description:** Execute the pipeline to verify that no multi-page document is artificially skipped, and the LLM correctly infers `is_continuation` using the footer hint.
**Target File:** `src/pipeline.py` (runtime)
**Autonomous:** false
**Actions:**
1. Run the pipeline against a test dossier containing multi-page correspondences with Arabic footers.
2. Verify via terminal logs that "Passing to LLM" correctly outputs the footer text hint.
3. Validate that the LLM infers `is_continuation=False` for page 1 ("1 of X") and `is_continuation=True` for subsequent pages ("2 of X").
4. Check the output directories to ensure the multi-page correspondences are grouped correctly into single PDF files without any lookahead bugs.
