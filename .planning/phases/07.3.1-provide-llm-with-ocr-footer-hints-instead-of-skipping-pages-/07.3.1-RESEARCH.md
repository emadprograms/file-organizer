# Phase 07.3.1 Research: Provide LLM with OCR footer hints

## Objective
Answer: "What do I need to know to PLAN this phase well?"

## Current State Analysis
1. **`src/pipeline.py` (Pass 1)**: The Mac Vision OCR currently scans the bottom 10% of each page for Arabic footers like "1 من 10". If detected, it entirely skips the LLM call and returns a fallback `PageClassification` with `is_continuation=True`. This is dangerous because page 1 ("1 of 10") is treated as a continuation and mistakenly merged with preceding unrelated documents.
2. **`src/llm.py`**: The `classify_page` method only takes `image_bytes` and does not accept or utilize OCR text hints. The system prompt does not explicitly demand `is_continuation` or `is_form` in its listed JSON keys instruction.
3. **`src/pipeline.py` (Pass 2)**: Contains `lookahead_continuation` logic that artificially groups pages if the *next* page is a continuation. This was a workaround for the blind `is_continuation` logic.

## Changes Needed for Implementation

### 1. `src/pipeline.py` (Vision OCR Logic in Pass 1)
- Modify the Vision OCR block inside `process_single_page`.
- Instead of setting `footer_detected = True` and returning early, extract the matched string:
  ```python
  extracted_footer = None
  # ... inside if success: ...
  for observation in request.results():
      top_candidate = observation.topCandidates_(1)
      if top_candidate:
          text = top_candidate[0].string()
          match = re.search(r'(\d+\s*من\s*\d+|الصفحة\s*\d+|صفحة\s*\d+)', text, re.IGNORECASE)
          if match:
              extracted_footer = match.group(1)
              break
              
  if extracted_footer:
      print(f" Page {p_idx} has Arabic footer '{extracted_footer}'. Passing to LLM.")
  ```
- Do not skip the LLM. Pass `extracted_footer` to the `classify_page` method:
  ```python
  res = self.client.classify_page(image_bytes=i_bytes, footer_text=extracted_footer)
  ```

### 2. `src/llm.py` (LLM Prompt & Method Signature)
- Update `classify_page` signature: `def classify_page(self, image_bytes: bytes, footer_text: str = None) -> PageClassification:`
- Dynamically augment the `user_prompt` in `classify_page`:
  ```python
  user_prompt = "Classify this scanned document page."
  if footer_text:
      user_prompt += f"\n\nHINT: The OCR system detected a page footer: '{footer_text}'. If this indicates it's the first page (like '1 من X'), it is NOT a continuation. If it indicates a subsequent page (like '2 من X' or 'الصفحة 2'), set is_continuation = true."
  ```
- Update `_build_system_prompt()` to explicitly include all schema fields at the bottom:
  ```text
  Return a JSON object with: house_number, residents (list of strings), category, date, is_continuation (boolean), is_form (boolean), and needs_gemma_fallback (boolean).
  ```

### 3. `src/pipeline.py` (Grouping Logic in Pass 2)
- Remove `lookahead_continuation`. The LLM will accurately mark page 2, 3, etc. as `is_continuation=True` based on the footer hint, so no lookahead heuristics are needed.
- Change the `effective_continuation` assignment to simply:
  ```python
  effective_continuation = getattr(page, 'is_continuation', False)
  ```

## Summary
By letting the LLM interpret the exact pagination string (e.g., distinguishing "1 of 10" from "2 of 10"), we avoid blind merging bugs. We can also clean up the messy `lookahead_continuation` heuristic.
