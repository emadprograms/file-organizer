### 1. `src/pipeline.py`
- **Role:** Orchestrates the multi-pass document processing and logic for page grouping.
- **Data Flow:** 
  - **Pass 1:** It uses Mac Vision OCR to extract text from the bottom 10% of each page. Instead of skipping the LLM when an Arabic footer is detected, it extracts the footer string and passes it into `classify_page` for the LLM to analyze.

### 2. `src/llm.py`
- **Role:** Communicates with the local LLM and the Gemini fallbacks to classify pages.
- **Data Flow:** Receives the `footer_text` hint from the pipeline and appends it to the user prompt. The updated system prompt ensures all standard schema fields are explicitly returned by the model.
- **Closest Existing Analog:** The existing `classify_page` method where prompts are sent to the local client (`self.local_client.beta.chat.completions.parse`) and the schema definitions at the end of `_build_system_prompt()`.
- **Code Excerpts:**

  **Prompt and Signature Updates:**
  ```python
  def classify_page(self, image_bytes: bytes, footer_text: str = None) -> PageClassification:
      system_prompt = self._build_system_prompt()
      user_prompt = "Classify this scanned document page."
      
      if footer_text:
          user_prompt += f"\n\nHINT: The OCR system detected a page footer: '{footer_text}'. If this indicates it's the first page (like '1 من X'), it is NOT a continuation. If it indicates a subsequent page (like '2 من X' or 'الصفحة 2'), set is_continuation = true."
  ```

  **System Prompt Schema Update:**
  ```text
  Return a JSON object with: house_number, residents (list of strings), category, date, is_continuation (boolean), is_form (boolean), and needs_gemma_fallback (boolean).
  ```
