# Phase 07.3.1 Context & Discussion

**User Request**:
The logic was supposed to be: the LLM still goes through the pages one by one but also looks for the footer. It doesn't blindly start skipping pages. You aren't supposed to skip. You still make the LLM call, but the LLM should know that there is a footer and should be given a basic choice to just group it with the previous one or not. Not skip entirely. Right now I'm branching the current read to OCR on the PC and text pass of the document to DeepSeek. So what we can do is that we can allow the local LLM to judge if it can be grouped with the previous one.

**Decisions Made**:
1. Revert the "bypass LLM entirely" logic added in 07.3.
2. Keep the Mac Vision OCR logic that scans the bottom 10% for the Arabic footer.
3. Instead of returning early and skipping the LLM, pass the detected footer string to the LLM prompt.
4. The local LLM will output the standard `PageClassification` schema, utilizing its built-in `is_continuation` flag based on the footer context provided to it.
5. In Pass 2, the lookahead logic should be removed/refined since the LLM itself will explicitly decide if it's a continuation based on the footer.

**Outcome**:
This ensures every page is correctly processed while still taking advantage of the pagination details, avoiding the edge cases where independent documents were blindly swallowed due to lookahead grouping.
