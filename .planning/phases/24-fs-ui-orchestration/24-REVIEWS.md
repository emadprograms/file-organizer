# Cross-AI Plan Review — Phase 24

## Antigravity Subagent Review

**Summary**
The plans correctly outline the class-based separation of the FSUI Orchestrator and the PID-based lock utility, fulfilling the phase requirements. However, there is a critical functional disconnect between the Orchestrator's `finalize` logic and the existing pipeline passes in `main.py`. The plans fail to account for the fact that `run_generation_pass` heavily relies on files generated by `process_unclassified_pdf` (specifically `_report.json` and `_categorized.pdf`), which are intentionally bypassed or suppressed in the `append` mode, guaranteeing runtime crashes upon finalization.

**Strengths**
- **Resilient Lock Strategy (24-01):** Utilizing `os.kill(pid, 0)` for stateless liveness checks accurately implements standard POSIX patterns and handles stale locks robustly.
- **Overwrite Protection (24-02):** Explicitly handling filename collisions during the final move (per D-13 constraint) is well captured in the `finalize` task.
- **Architectural Encapsulation (24-02):** Replacing the procedural loop in `src/main.py:81` with the `FSUIOrchestrator` class properly isolates the polling state and logic (FSUI-06).

**Concerns**
- **HIGH:** Missing OCR payload for Pipeline Passes (`24-02-PLAN.md`, `finalize`). The plan dictates running `run_cleaning_pass`, which strictly requires a `_report.json` file as its first argument (`json_path`). If a file is fully specified in its name (no 'U' fields), `infer_missing_data` (in `src/inbox/resolver.py:18`) never invokes `process_unclassified_pdf`, meaning `_report.json` is never generated. The pipeline passes will fail to execute without it.
- **HIGH:** Hardcoded dependency on `_categorized*.pdf` (`24-02-PLAN.md`, `finalize`). The orchestrator calls `run_generation_pass` (`src/main.py:341`), which blindly attempts to find `list(target_dir.glob(f"{house_id}_categorized*.pdf"))[0]`. However, `append` mode suppresses the creation of this file (`create_categorized_copy=False` in `src/inbox/resolver.py:24`), guaranteeing an `IndexError` crash during finalization.
- **MEDIUM:** Potential memory leak in stateless loop (`24-02-PLAN.md`, `process_inbox`). The plan states "Track file sizes in a local dict" for size stability (D-02). If the dictionary tracks all files—including ignored `_Proposed` and ` OK` suffixes—it will grow indefinitely during long polling sessions.
- **LOW:** Implicit strict pathing (`24-02-PLAN.md`). The threat model (T-24-02-01) requires strict path resolution for file moving, but the action text for `finalize` does not explicitly mandate using `pathlib.Path.resolve(strict=True)`.

**Suggestions**
- **Synthesize or mandate OCR during `finalize`:** If `append` mode relies on the filename for all metadata, either synthesize a mock `_report.json` based on the parsed filename, or explicitly trigger `process_unclassified_pdf` inside `finalize` if `_report.json` is missing.
- **Fix `run_generation_pass` file lookup:** Modify `src/main.py:341` to accept the finalized filename directly rather than enforcing the `_categorized*.pdf` glob pattern, or ensure `finalize` passes the correct PDF path to the generation pass.
- **Scope the size-tracking dictionary:** Explicitly state in the `process_inbox` task that the local size-tracking dictionary will only track files that do *not* have `_Proposed`, `_Failed`, `_Error`, or ` OK` suffixes to prevent memory bloat over long sessions.
- **Enforce strict path resolution:** Explicitly require `.resolve(strict=True)` in `FSUIOrchestrator.finalize` when calculating the final destination path to mitigate path traversal risks (T-24-02-01).

**Risk Assessment**
HIGH. The current plan for `finalize` guarantees a crash due to structural incompatibilities with how the `main.py` pipeline functions (`run_cleaning_pass` and `run_generation_pass`) expect their inputs—specifically the absence of `_report.json` and `_categorized.pdf` in the append workflow.
