  <files>src/fs_ui/__init__.py, src/fs_ui/lock.py, tests/test_fs_ui_lock.py</files>
  <behavior>
    - Test 1: `acquire_lock` successfully writes the current PID to a new lockfile.
    - Test 2: `acquire_lock` raises `LockExistsError` if the lockfile contains a PID that is currently alive.
    - Test 3: `acquire_lock` silently overwrites the lockfile if it contains a PID that is no longer alive.
    - Test 4: `release_lock` removes the lockfile only if it exists and matches the current PID.
  </behavior>
  <action>
    Create `src/fs_ui/__init__.py` (empty or basic package structure).
    Create `src/fs_ui/lock.py` containing a custom `LockExistsError` exception and two functions: `acquire_lock(lock_path: Path)` and `release_lock(lock_path: Path)`.
    `acquire_lock` should check if `lock_path` exists. If so, read the PID. Use `os.kill(pid, 0)` (ignoring OSError) to check if the process is alive. If alive, raise `LockExistsError`. If dead, overwrite the lockfile with the current process's `os.getpid()`.
    Use `LockExistsError` so the orchestrator can catch it and exit cleanly.
  </action>
  <verify>
    <automated>pytest tests/test_fs_ui_lock.py -x</automated>
  </verify>
  <done>The PID lock utility successfully acquires fresh locks, rejects conflicts from alive PIDs, and recovers stale locks from dead PIDs.</done>
