# Phase 19.1.1.1 Context

## Domain
Update `yaml_loader.py` to check the `.source_files/` directory for the YAML configuration instead of the root `target_dir`.

## Canonical References
- `src/tenant_config/yaml_loader.py`

## Decisions Captured

### YAML Filename and Location
- **Decision**: The pipeline must look for exactly `target_dir / .source_files / {house_id}_tenants.yaml`.
- **Details**: The `.source_files/` folder will be located inside the `target_dir` (the folder being organized). The filename will remain `{house_id}_tenants.yaml`.
- **Reasoning**: This keeps the house-specific naming convention intact while safely isolating user-provided source files in a dedicated hidden directory.

### Multiple YAMLs Handling
- **Decision**: Look for the exact filename only.
- **Details**: Since we are targeting a specific `{house_id}_tenants.yaml` filename, any other `.yaml` files in the `.source_files/` directory should be safely ignored.
