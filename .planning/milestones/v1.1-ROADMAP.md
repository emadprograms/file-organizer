---

## Milestone Summary

**Key Decisions:**
- Decision: Changed pipeline concurrency to sequential (1 worker) due to strict 15 RPM IP limit, while retaining multi-key daily quota rotation.
- Decision: Implemented exponential backoff with jitter to prevent cascading 429 errors into 500 server errors.
- Decision: Added logic to abort page retries after 2 invalid LLM responses to avoid wasting rate-limit slots.
- Decision: Enforced global 15 RPM cap across all keys combined.

**Issues Resolved:**
- Silent failures on 429s causing empty folders.
- Lock contention across processes on API limits.
- Wasted tokens on unparseable/invalid JSON responses.
