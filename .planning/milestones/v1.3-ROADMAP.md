# Milestone v1.3 Roadmap: Routing Decoupling & Checkpointing

**Status:** Completed (with known integration gaps)
**Phases:** 11, 12, 13

## Objective
Decouple the routing step from grouping to improve pipeline resilience, enable independent LLM model configuration, and support resuming routing without re-running grouping. Also, implement conditional LLM folder routing with 1:1 English-to-Arabic mapping.

## Key Accomplishments

### Phase 11: Conditional LLM Folder Routing and Folder Renaming
- **Arabic Folder Mapping**: Implemented strict 1:1 mapping of categories to Arabic folder names.
- **Constrained Routing**: Introduced constrained prompting and a "double-check" mechanism for miscellaneous documents to reduce hallucinations.
- **System-Wide Renaming**: Migrated all destination folder paths from English to Arabic across the codebase.

### Phase 12: Finalize Conditional LLM Folder Routing and Folder Renaming
- **Configuration Unification**: Centralized the `DIRECT_ROUTING_MAP` in `src/processing/routing/config.py` and removed redundant local maps in the router.
- **Zero-English Audit**: Performed a final system-wide scan to ensure no hardcoded English destination paths remain in `src/`.

### Phase 13: Routing Checkpoints & Architecture Decoupling
- **Pipeline Decoupling**: Refactored `src/processing/pipeline.py` to separate the routing process into its own loop after grouping is finalized.
- **Granular Checkpointing**: Implemented `RoutingStateManager` to save state atomically after every single document is routed.
- **Grouping Checksum**: Introduced a checksum-based sanity check to prevent routing resumption on inconsistent grouping states.
- **Dynamic Model Configuration**: Updated the routing API to support independent LLM model selection for the routing stage.

## Verification Summary
- **Phase Tests**: All phases passed their respective unit and integration tests.
- **E2E Flow**: Verified that documents are correctly routed into Arabic folders.
- **Audit Finding**: A critical integration gap was found during the milestone audit: the routing checkpoint mechanism tracks progress but does not persist the routing results, causing data loss upon resumption.
