# Phase 02 Plan 02: Fix Module-Level Environment Checks

**Goal:** Close gaps diagnosed in UAT by deferring environment variable checks until `load_config` is called, ensuring `load_dotenv` in `main.py` has a chance to execute first.

## Context
In `src/config.py`, `os.environ.get()` is evaluated at the module level. This causes a fail-fast crash when the application starts because `src.config` is imported in `main.py` before `load_dotenv()` is called.

## Gap Addressed
- **Test 1 (Cold Start Smoke Test):** Server crashes immediately on start with 'FATAL ERROR: Missing required API keys in environment.'

## Tasks

### Task 1: Defer Environment Check in `src/config.py`
- Open `src/config.py`.
- Remove the module-level evaluation of `gemini_key`, `openrouter_key`, `groq_key` and the `if` block at the bottom of the file.
- Ensure `load_config()` retains its own checks and throws the FATAL ERROR if variables are missing.

### Task 2: Ensure `main.py` correctly handles configuration
- Open `src/main.py`.
- Verify `load_dotenv()` runs before `load_config()`.
