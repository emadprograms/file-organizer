# Phase 02: Key Management & Configuration

## Goal
Simplify API key loading and switching, and enforce strict environment configuration validation.

## Requirements
- REF-03

## must_haves
- truths:
  - `src/config.py` contains logic that requires `GEMINI_API_KEY`, `OPENROUTER_API_KEY`, and `GROQ_API_KEY` to be present at startup.
  - The application exits immediately (fails fast) if any required key is missing.
  - Quota tracking logic exists and uses a tracking directory with a permanent file to log successful call timestamps and enforces a limit of 1500 calls per 24 hours.
  - Startup calculates and displays the remaining quota out of 1500.
  - `src/llm.py` enforces a strict minimum of 7 seconds per call.
  - `src/llm.py` waits 65 seconds before retrying on a `429` error.
  - `src/llm.py` waits 15 seconds before retrying on a `500` or `503` error.
- prohibitions: []

## Artifacts this phase produces
- `src/config.py` module
- `src.config.load_config` function
- `src.config.record_successful_call` function
- `.quota/` or `.tracking/` directory containing the quota log file
